What is ChatBox?

ChatBox is a comprehensive messaging platform built with Flutter that combines enterprise-grade reliability with consumer-grade user experience. Whether you're building team communication tools, social messaging apps, or customer support platforms, ChatBox provides all the building blocks you need.

üí° Pro Tip: ChatBox is designed with offline-first architecture, ensuring your users can continue messaging even when connectivity is poor. Messages are automatically synchronized when the connection is restored.

üöÄ Production Ready

Built for scale with enterprise-grade infrastructure. Handles thousands of concurrent users with real-time message delivery, presence indicators, and intelligent caching.

  • 99.9% uptime with automatic failover
  • Global CDN for low-latency messaging
  • End-to-end encryption for secure communications
  • Advanced monitoring and analytics
üì±
‚ö°

Real-Time Messaging

Instant message delivery with WebSocket connections, typing indicators, read receipts, and seamless offline synchronization. Supports rich media, reactions, and threaded conversations.

üîê

End-to-End Encryption

Military-grade encryption using ECDH key exchange and AES-256. Messages are encrypted before transmission and can only be decrypted by intended recipients.

üé§

Voice Integration

Advanced speech-to-text with multi-language support, voice message recording, and hands-free messaging capabilities. Integrated with Google Cloud Speech API.

üîÑ

Offline First

Messages queue automatically when offline and sync seamlessly when connection is restored. SQLite database ensures data persistence across app restarts.

üé®

Beautiful Animations

Delightful micro-interactions with confetti celebrations, smooth transitions, and Flutter Animate integration. Customizable animation themes and durations.

Strongly typed models with comprehensive error handling, ensuring robust applications with fewer runtime errors. Full null safety support.

User Authentication

Secure authentication with Firebase Auth supporting email/password, Google Sign-In, and biometric authentication. JWT tokens with automatic refresh.

Customizable UI

Fully customizable themes, message bubbles, and UI components. Support for dark/light modes, custom fonts, and brand colors.

Built-in performance monitoring, user engagement tracking, and detailed message analytics for data-driven insights.

Quick Installation

1

Clone Repository

Get the latest version of ChatBox from GitHub

git clone https://github.com/Umerjamshaid/ChatBox-App1.2.git cd ChatBox-App1.2
2

Install Dependencies

Install all required Flutter packages and dependencies

flutter pub get flutter pub run build_runner build --delete-conflicting-outputs
3

Configure Environment

Set up your API keys and configuration files

# Create environment configuration cp .env.example .env # Edit .env with your API keys
4

Run Application

Launch the app in development mode

flutter run # Or for web flutter run -d chrome
‚ö†Ô∏è Important: Make sure to configure your Firebase project correctly and add your google-services.json (Android) or GoogleService-Info.plist (iOS) files to the respective platform directories.

Firebase Configuration

Configure Firebase Authentication and Firestore for user management and data persistence.

1

Create Firebase Project

Go to Firebase Console and create a new project

2

Enable Authentication

Enable Email/Password and Google sign-in methods

3

Setup Firestore

Create Firestore database in test mode

4

Download Config Files

Add google-services.json and GoogleService-Info.plist to your project

Stream Chat Integration

Set up Stream Chat for real-time messaging capabilities.

// Initialize Stream Chat client final client = StreamChatClient( 'YOUR_API_KEY', logLevel: Level.INFO, ); // Connect user await client.connectUser( User(id: userId, name: userName), userToken, );

Environment Variables

Configure your environment variables for different deployment environments.

# .env file STREAM_API_KEY=your_stream_api_key FIREBASE_PROJECT_ID=your_project_id GOOGLE_CLOUD_API_KEY=your_google_cloud_key ANALYTICS_KEY=your_analytics_key
